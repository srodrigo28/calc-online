Índice do Piloto

    Plano Final Detalhado para o Teste Piloto

    Uma análise refinada dos modelos de dados, telas e sequência de criação.

    1. Definição Final de Campos (O Modelo de Dados)

    Esta é a estrutura de dados detalhada e otimizada para o piloto, focada em coletar as informações essenciais e minimizar o atrito no cadastro.

    Tabela `profiles` (Usuários e Vendedores)

    Nome do Campo Tipo de Dado Obrigatório? Descrição e Recomendações
    id UUID ✅ Sim Chave primária (FK de `auth.users`).
    email Texto ✅ Sim Único por usuário.
    full_name Texto ✅ Sim Nome completo, essencial no cadastro.
    is_author Booleano ✅ Sim Define o papel. Padrão: false.
    bio Texto Longo ❌ Não Biografia do autor (preenchido pós-cadastro).
    avatar_url Texto ❌ Não URL para a foto de perfil.

    ⭐ Recomendação Estratégica Importante:

    Pedir muitos dados no cadastro (telefone, cidade, etc.) diminui drasticamente a taxa de conversão. A melhor estratégia é: 1. No cadastro, peça apenas o essencial (Nome, Email, Senha). 2. No Dashboard, incentive o usuário a completar o perfil para uma melhor experiência.

    Tabela `books` (Obras Publicadas)

    A estrutura para os livros permanece a mesma, otimizada para performance ao incluir o nome do autor (denormalização).

    Nome do Campo Tipo de Dado Obrigatório? Descrição
    idUUID✅ SimChave primária.
    author_idUUID✅ SimConecta com o autor em `profiles`.
    author_nameTexto✅ SimNome do autor (denormalizado).
    titleTexto✅ SimTítulo da obra.
    descriptionTexto Longo✅ SimSinopse do livro.
    cover_urlTexto✅ SimURL da imagem da capa.
    categoryTexto✅ SimCategoria principal.
    priceNumérico✅ SimPreço pretendido em BRL.
    statusTexto✅ Sim'publicado' ou 'rascunho'.

    2. Detalhes Finais das Telas do Piloto

    Com o modelo de dados definido, refinamos o escopo funcional de cada tela essencial para o piloto.

    Tela 1: Home/Vitrine

    • Exibição dos livros em grid (capa e título).
    • Barra de busca por título, categoria ou autor.
    • Filtros visíveis por categoria (ex: botões de filtro).
    • Clique em um livro leva para a `PaginaDetalheLivro`.

    Tela 2: Página de Detalhes do Livro

    • Layout de duas colunas (capa na esquerda, informações na direita).
    • Exibição de todos os dados do livro: capa, título, autor, categoria, descrição, preço.
    • **Nova Seção "Sobre o Autor"**: Abaixo dos detalhes, exibir a foto, nome e biografia do autor para criar conexão.
    • **Métrica Chave (CTA):** Botão "🔥 Tenho Interesse!" que, para usuários logados, registra a ação na tabela `interesses`.

    Tela 3: Autenticação

    • Interface com "flip card" animado para Login e Cadastro.
    • **Cadastro Simplificado:** Solicitar apenas Nome Completo, Email e Senha.
    • Oferecer login social (Google) como opção de menor atrito.
    • Após o primeiro login, incentivar o preenchimento do perfil no dashboard.

    Tela 4: Dashboard

    • **Componente Chave:** Seletor de modo "Modo Leitor" vs. "Modo Autor".
    • **Visão Leitor:** Lista de livros com interesse demonstrado.
    • **Visão Autor:** Cards de stats ("Livros Publicados", "Interesses Recebidos") e uma tabela para gerenciar (Editar/Excluir) as obras.
    • **Seção "Meu Perfil":** Área para o usuário preencher/editar os dados opcionais (bio, avatar, etc.).

    3. Nova Sequência de Criação (Otimizada)

    Esta é a ordem de desenvolvimento recomendada para construir o piloto de forma incremental e focada.

    1. Fundação e Autenticação:
      • Configurar o projeto Next.js e conectá-lo ao Supabase.
      • Criar o schema do banco de dados (tabelas `profiles`, `books`, `interesses`) com as políticas de segurança.
      • Construir a tela de Autenticação (Login/Cadastro).
      • Criar o Header Dinâmico que reflete o estado da sessão.
    2. O Lado do Vendedor:
      • Construir a estrutura do Dashboard com o seletor de modo.
      • Focar na Visão "Modo Autor", implementando o CRUD de livros com upload de capa.
      • Exibir os stats simples.
    3. A Exposição Pública:
      • Construir a Home/Vitrine para listar os livros.
      • Criar a Página de Detalhes do Livro.
      • Implementar a lógica do botão "Tenho Interesse!".
    4. Fechando o Ciclo:
      • Implementar a Visão "Modo Leitor" no Dashboard.
      • Fazer o polimento final da interface e responsividade.
      • Lançar o teste para os "Pioneiros" e começar a coletar dados.
