Roteiro do Piloto

    Plano de Construção: Teste Piloto E-Leitor

    Um roteiro enxuto para validar o marketplace de livros digitais.

    1. Conceito e Telas Essenciais

    O objetivo do piloto é validar o interesse de autores e leitores em um novo marketplace de nicho, antes de construir funcionalidades complexas como o leitor seguro e o checkout financeiro.

    Hipóteses a Validar:

    1. Autores independentes têm interesse em cadastrar suas obras em uma nova plataforma focada em seu nicho?
    2. Leitores demonstrarão interesse de compra/leitura pelas obras catalogadas?

    Telas Essenciais para o Piloto:

    • Home/Vitrine: Exibe os livros com busca simples por título/autor.
    • Página do Livro: Detalhes completos da obra com um botão "Tenho Interesse!".
    • Autenticação: Tela única de Login/Cadastro.
    • Dashboard: Painel unificado com modo "Leitor" (lista de interesses) e modo "Autor" (gerenciador de livros).
    • Formulário do Livro: Tela para o autor criar/editar os dados do livro e fazer upload da capa.

    2. Etapa 1: Fundação e Autenticação

    Objetivo: Ter um sistema onde usuários possam se cadastrar, logar e ter seus perfis criados.

    Definição do Banco de Dados Simplificado:

    Crie as tabelas abaixo no Supabase. O schema já inclui as relações e políticas de segurança (RLS) essenciais para o piloto.

    -- Tabela 'profiles' para dados de usuário (vendedor/leitor)
CREATE TABLE public.profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  full_name TEXT,
  is_author BOOLEAN DEFAULT false,
  is_pioneer BOOLEAN DEFAULT false -- Para o "Programa Pioneiros"
);
-- Habilita RLS
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
-- Policies para a tabela profiles (ex: usuário só pode editar seu próprio perfil)
CREATE POLICY "Users can view all profiles." ON public.profiles FOR SELECT USING (true);
CREATE POLICY "Users can update their own profile." ON public.profiles FOR UPDATE USING (auth.uid() = id);

-- Tabela 'books' para os metadados dos livros
CREATE TABLE public.books (
  id UUID PRIMARY KEY DEFAULT extensions.uuid_generate_v4(),
  author_id UUID NOT NULL REFERENCES public.profiles(id),
  title TEXT NOT NULL,
  category TEXT,
  description TEXT,
  price NUMERIC(8, 2) NOT NULL,
  cover_url TEXT, -- Path no Supabase Storage
  created_at TIMESTAMPTZ DEFAULT now()
);
-- Habilita RLS
ALTER TABLE public.books ENABLE ROW LEVEL SECURITY;
-- Policies para a tabela books (ex: todos podem ver, mas só o autor pode editar/deletar)
CREATE POLICY "Books are viewable by everyone." ON public.books FOR SELECT USING (true);
CREATE POLICY "Authors can insert their own books." ON public.books FOR INSERT WITH CHECK (auth.uid() = author_id);
CREATE POLICY "Authors can update their own books." ON public.books FOR UPDATE USING (auth.uid() = author_id);
CREATE POLICY "Authors can delete their own books." ON public.books FOR DELETE USING (auth.uid() = author_id);

-- Tabela 'interesses' para validar a demanda
CREATE TABLE public.interesses (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES public.profiles(id),
  book_id UUID NOT NULL REFERENCES public.books(id),
  created_at TIMESTAMPTZ DEFAULT now(),
  CONSTRAINT "interesses_user_book_unique" UNIQUE (user_id, book_id)
);
-- Habilita RLS
ALTER TABLE public.interesses ENABLE ROW LEVEL SECURITY;
-- Policies para a tabela interesses (usuário só pode gerenciar seu próprio interesse)
CREATE POLICY "Users can manage their own interests." ON public.interesses FOR ALL USING (auth.uid() = user_id);

    ⚖️ Ponto de Atenção (Privacidade):

    Desde o início, sua aplicação coletará dados de usuários. Garanta que o formulário de cadastro inclua um checkbox para aceitar os Termos de Serviço e a Política de Privacidade.

    3. Etapa 2: O Lado do Vendedor

    Objetivo: Permitir que os "Pioneiros" publiquem e gerenciem suas obras.

    Tarefas Principais:

    • Construir a página do Dashboard com o seletor de modo "Leitor/Autor".
    • Na visão "Autor", implementar o CRUD (Criar, Ler, Atualizar, Deletar) de livros.
    • O formulário de criação deve permitir o upload da imagem de capa para o **Supabase Storage**.
    • Exibir os stats simples: "Total de Livros Publicados" e "Total de Interesses Recebidos" (contando registros na tabela `interesses` para os livros do autor).

    🛡️ Ponto de Atenção (Segurança):

    Configure as políticas do Supabase Storage para que um autor só possa fazer upload de capas para seus próprios livros. Isso geralmente é feito criando pastas com o `user_id` do autor no nome (ex: `covers/user-abc-123/minha-capa.png`).

    4. Etapa 3: Exposição Pública e Validação

    Objetivo: Expor as obras cadastradas e medir o interesse do público leitor.

    Tarefas Principais:

    • Construir a `HomeVitrine` que busca todos os livros da tabela `books` e os exibe.
    • Implementar a busca simples que filtra os livros por título ou autor.
    • Programar o botão **"Tenho Interesse!"** na página de detalhes do livro.

    🔄 Ponto de Atenção (Robustez):

    A lógica do botão "Tenho Interesse!" deve ser idempotente: um usuário só pode registrar interesse em um mesmo livro uma única vez. A constraint `UNIQUE` que adicionamos na tabela `interesses` já garante isso no nível do banco de dados, protegendo contra cliques duplicados.

    5. Etapa 4: Fechando o Ciclo do Piloto

    Objetivo: Finalizar a experiência do usuário leitor e preparar para o lançamento do teste.

    Tarefas Principais:

    • Na visão "Modo Leitor" do Dashboard, listar os livros em que o usuário demonstrou interesse (fazendo um `JOIN` entre `interesses` e `books`).
    • Realizar o polimento final de toda a interface, garantindo a responsividade.
    • Lançar o teste, convidando os "Pioneiros" e começando a analisar os dados da tabela `interesses` para validar as hipóteses.
